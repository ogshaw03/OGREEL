## Firestore 永続キャッシュの挙動（2026-06-06 記録）

dev に導入した **オフライン永続キャッシュ（IndexedDB）** の挙動まとめ。

実装：
```js
const db = initializeFirestore(app, {
  localCache: persistentLocalCache({ tabManager: persistentMultipleTabManager() })
});
```

### キャッシュが「残る」ケース（再利用され、Reads を抑え続ける）
- PC再起動 / ブラウザ再起動 / タブを閉じる → そのまま保持
- 次回ロード時：**キャッシュから即表示 → サーバーには差分のみ取得**
- IndexedDB はディスク保存のため、セッションをまたいで永続する

### キャッシュが「消える」ケース（次回は初回ロード扱い＝全件1回読み直し）
- ブラウザで「サイトデータ／Cookie・キャッシュの削除」を実行
- シークレット／プライベートモード（セッション終了時に破棄）
- 別のブラウザ・別端末・別プロファイル（キャッシュはブラウザ×オリジン単位）
- ディスク容量逼迫時のブラウザによる自動退避（まれ）

### 消えても問題ない理由
- データの正本は常に Firestore 側にあるため、**キャッシュが消えても失われるデータはない**
- 消えた次の1回だけ全件読み込み（＝初回と同じ）→ 以降また自動でキャッシュ再構築
- 古いキャッシュが残っていても、接続時にサーバーと照合して最新へ更新される（古い内容が出続けることはない）

### 結論
通常運用ではキャッシュは生き続け、リロード起因の Reads を継続的に抑制する。
効果が一時的にリセットされるのは「サイトデータ削除」「シークレットモード」等の限られたケースのみ。
